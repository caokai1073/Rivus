# 我做了一个完全本地运行的 AI 知识库工具，开源了

## 起因：我攒了一堆"以后要看"的东西，但从来找不到

我有一个坏习惯：遇到好文章就收藏，遇到有用的 PDF 就下载，遇到值得记下来的想法就扔进备忘录。

时间久了，浏览器书签几百条，下载文件夹乱成一锅粥，备忘录里全是零散笔记。每次真正需要找什么的时候，脑子里只记得"好像存过这个"，但就是找不到。

后来我试过很多工具——Notion、Obsidian、各种 RAG demo——但总有一个问题困扰我：**我不想把私人文档传到别人的服务器上。** 尤其是工作相关的资料、个人笔记，传到云端总觉得不踏实。

所以我花了一段时间，自己做了一个：**Rivus · 问渠**。刚刚开源，来分享一下。

---

## 它是什么

一句话：**把你的文档喂给 AI，然后用自然语言提问。全程本地，零数据上传。**

你可以把 PDF、Word、Excel、Markdown、网页 URL、手打的笔记全部扔进去，然后像问朋友一样问它问题。它会找到相关内容，给你一个有根据的回答，还附上来源链接。

不需要 OpenAI Key，不需要网络，不需要订阅。模型跑在你自己的机器上。

---

## 具体怎么用

打开 app，左边是知识库，右边是对话框。

**添加文档**很简单：
- 拖进去一个 PDF，它自动解析
- 粘贴一个网页链接，它去抓取正文
- 上传 Excel 表格，它把每个 sheet 都转成可检索的文本
- 直接在输入框打字，保存成一条笔记

**提问**就像发消息：

> "上个月那篇关于 RAG 评估方法的论文，作者是谁，结论是什么？"

> "我有哪些关于肠道菌群的资料？"

> "帮我整理一下关于 TCR 识别机制的几篇文献，主要观点有什么不同？"

它会自动在你的知识库里搜索，把相关段落找出来，综合成一段回答。回答下面会附上来自哪篇文档的 citation，点击直接打开原文。

## 技术上它是怎么工作的

这部分给感兴趣的朋友说一下。

**检索部分**用的是混合 RAG：
- 向量检索（`BAAI/bge-m3` 嵌入模型，支持中英文混合）
- BM25 全文检索
- RRF（Reciprocal Rank Fusion）融合两路结果

光是"把问题拆成几个子问题再分别检索"这一步，就让检索质量提升了很多。比如你问"这个化合物的毒性和代谢途径"，它会拆成"毒性研究"和"代谢途径"两个方向分别去找，然后合并结果。

**本地模型**走 Ollama，Qwen3、Llama3、Gemma3 都支持，一键下载，不用碰终端。

**云端模型**也支持——DeepSeek、OpenAI、Claude、智谱 GLM 都可以接。用云端模型时，走的是 tool-use agent：模型自己决定要搜几次、搜什么，直到觉得信息够了才回答。这比固定的"检索 top-k"逻辑要聪明不少。

**存储**只用 SQLite + sqlite-vec，整个知识库就是一个 `.db` 文件，随便备份、随便迁移。

## 我每天怎么用它

我主要用它做三件事（模型用的是通过 SSH 隧道连接实验室服务器上的 Qwen3-72B，笔记本本身跑不起来这个量级）：

**1. 文献管理代替品**

我做生物信息学，手边有几十篇 PDF 论文。以前查一个方法细节要挨个打开翻，现在直接问："哪篇文章里提到了用 MMseqs2 聚类的参数设置？" 几秒钟出来。

**2. 个人知识沉淀**

遇到有意思的技术文章，以前是扔进 Pocket 然后永远不再看。现在我直接把 URL 粘进 Rivus，它自动抓取正文存起来。几个月下来攒了一两百条，全都可以检索，感觉像是在和自己过去的阅读记录对话。

**3. 会议/项目笔记**

项目里讨论过的决定、设计思路，我会用手打笔记的方式存进去。以后想起"当时为什么这么决定"，直接问就行，不用翻聊天记录。

## 一些小细节，但我很喜欢

- **右键菜单**：对每个文档右键，可以直接打开原文件、下载、改名、批量选中删除
- **引用模式**：想针对某篇具体文档提问，右键"引用"钉住它，AI 就只从这篇里找答案
- **批量删除**：右键 → 选择，勾选多条一起删，Esc 取消
- **完全离线**：用本地模型时，断网也能跑，飞机上照用

## 隐藏功能：SSH 远程服务器模式

这个功能我觉得很多人会需要，但可能没想到。

场景是这样的：你有一台实验室服务器，或者家里有一台带 GPU 的台式机，跑 70B 级别的大模型完全没问题。但你平时用的是笔记本，算力不够。

**Rivus 支持通过 SSH 隧道连接远程 Ollama 实例。** 设置只需要三步：

1. 在远程机器上装好 Ollama，跑起来
2. 在 Rivus 设置里填入 SSH 主机地址、用户名、密码或密钥
3. 点击连接

之后 Rivus 会在后台自动建立 SSH 隧道，把远程的 Ollama 端口转发到本地。你在笔记本上提问，实际推理在服务器上跑，延迟低，模型质量高。

对于做研究的用户来说，这个功能尤其实用——实验室服务器往往有 A100/H100，跑 Qwen3-72B 或 Llama3-70B 效果远好于笔记本上的小模型，但以前要用只能 SSH 进去敲命令行。现在直接在 Rivus 里切换一下就行，界面和本地模式完全一样。

支持密码登录和 SSH 密钥两种认证方式，连接状态实时显示，断线会自动提示。
